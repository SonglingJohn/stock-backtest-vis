# A股回测工作台 — 项目使用手册

## 项目简介

**A股回测工作台** 是一款在本地运行的 A 股投资策略回测工具。你可以配置「买哪些股票、怎么买、买多少、从哪年到哪年」，系统会按历史行情模拟持仓市值变化，并以**动画回放**的方式展示收益曲线；也可跳过动画直接查看最终结果。

典型问题：*「如果我在 2014 年一次性买入贵州茅台 10 万元，持有至今能赚多少？」* 或 *「每年定投某几只股票，和沪深 300 相比表现如何？」*

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端运行时 | **Python 3** | 建议 3.10+ |
| HTTP 服务 | **标准库** `http.server.ThreadingHTTPServer` | 无 Flask/FastAPI 等框架依赖 |
| 行情数据 | **[AKShare](https://github.com/akfamily/akshare)** `>= 1.16.0` | 拉取 A 股日线、指数、名称列表 |
| 回测引擎 | 纯 Python（`backend/backtest.py`） | 无网络依赖，便于单元测试 |
| 数据缓存 | 本地 JSON（`.cache/`） | 减少重复请求，离线时可复用已缓存区间 |
| 前端 | **原生 HTML / CSS / JavaScript（ES Module）** | 无 React/Vue 构建链 |
| 图表 | **SVG** | 估值曲线、多标的对比 |
| 后端测试 | **`unittest`** | `tests/test_backtest.py`、`tests/test_data_source.py` |
| 前端测试 | **Node.js** `node:test` | `tests/*.test.mjs`（需本机安装 Node） |

### 价格与回测规则（实现约定）

- **个股价格**：后复权收盘价（`adjust="hfq"`），缓存文件名带 `hfq` 后缀。
- **一次性买入**：在回测**开始年份**的**首个交易日**按收盘价买入。
- **年定投**：在区间内**每一年**的**首个交易日**各投入设定金额。
- **结果截止日**：使用数据源在该区间内**最后一个交易日**，而非自然日「今天」。
- **多只股票**：按配置比例合并为一条「用户配置」组合曲线；单只股票时只显示该股票曲线。
- **宽基指数**：可拖入图表区作为基准对比（使用指数收盘价序列，不参与个股仓位比例）。

## 界面使用说明

### 左侧参数面板

#### 股票配置

- 每行可填 **6 位代码**（如 `600519`）或 **股票名称**（如 `贵州茅台`），失焦或按 Enter 会自动识别。
- **仓位比例**：多只股票时可填百分比；留空时单股视为 100%，多股未填则按 100% 参与归一化分配。
- **`+`**：增加一行；**`-`**：删除该行（至少保留一行）。
- 识别成功后会显示「行情已就绪」等状态；首次拉取数据需联网，之后会写入 `.cache`。

#### 买入假设

| 选项 | 含义 |
|------|------|
| **一次性买** | 仅在开始年首个交易日投入「金额」 |
| **年定投** | 区间内每年首个交易日各投入「金额」 |
| **金额** | 一次性总额，或每年定投额（元） |
| **回测区间** | 点击「回测区间」展开开始/结束年份；识别股票后可能根据上市年份建议开始年 |

#### 操作按钮

| 按钮 | 作用 |
|------|------|
| **播放回测** | 提交回测并**按年动画回放**曲线；回放时左侧面板可收起 |
| **直接看结果** | 跳过动画，直接显示区间末尾结果与汇总 |
| **展开参数 / ‹** | 回放过程中收起或展开左侧配置 |

#### 宽基指数（面板底部）

展示上证、深证成指、创业板、沪深 300、科创 50、北证 50 等卡片（数据来自 AKShare，失败时使用页面内置占位值）。

- **拖拽**指数卡片到右侧图表上方的「拖放区」，可与股票曲线一起对比。
- 已选指数以标签显示，点击 **×** 可移除。

### 右侧图表区

- **播放中**：顶部显示当前年份、累计投入、收益率、浮盈；曲线随时间推进。
- **暂停**：暂停动画。
- **时间轴滑块**：回放结束后可拖动查看不同时间段视口；**看整体** 显示完整区间曲线。
- **最终结果浮层**：回放结束或直接看结果时，展示各标的汇总（投入、市值、收益、年化、最大回撤等）。

### 典型操作流程

1. 启动 `python -m backend.server`。
2. 浏览器打开 `http://127.0.0.1:8000`。
3. 输入股票代码或名称，设置买入方式、金额与年份区间。
4. （可选）拖入宽基指数作基准。
5. 点击 **播放回测** 或 **直接看结果**。
6. 根据曲线与底部/浮层文字判断策略表现。

## HTTP API 参考

除静态页面外，后端提供以下 JSON 接口（均来自 `backend/server.py`）。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 服务是否正常 |
| GET | `/api/resolve-stock?q=贵州茅台` | 按代码或名称解析股票，返回 `symbol`、`name` |
| GET | `/api/stock-info?symbol=600519` | 股票名称、上市日期等 |
| GET | `/api/preload-stock?symbol=600519&startYear=2014&endYear=2026` | 预拉取并缓存行情，返回行数与首尾日期 |
| GET | `/api/market-indexes` | 宽基指数最新点位列表 |
| POST | `/api/backtest` | 执行回测，请求体见下表 |

### POST `/api/backtest` 请求体示例

```json
{
  "stocks": [
    { "symbol": "600519", "percent": "100" }
  ],
  "benchmarks": ["000300"],
  "buyMethod": "lump_sum",
  "amount": 100000,
  "startYear": 2014,
  "endYear": 2026
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `stocks` | 数组 | 至少一项；`symbol` 为 6 位代码；`percent` 可选 |
| `benchmarks` | 字符串数组 | 指数代码，如 `000300` |
| `buyMethod` | 字符串 | `lump_sum`（一次性）或 `annual`（年定投） |
| `amount` | 数字 | 金额，须 > 0 |
| `startYear` / `endYear` | 整数 | 回测年份闭区间 |

成功时返回 `series` 数组，每项含 `snapshots`（逐日市值序列）与 `summary`（汇总指标）。

## 数据缓存

- 个股：` .cache/prices/{代码}-{开始年}-{结束年}-hfq.json `
- 指数：` .cache/prices/index-{代码}-{开始年}-{结束年}.json `
- 股票名称表：` .cache/stock-names.json `
- 指数快照：` .cache/market-indexes.json `

若网络不可用但所需区间**已有缓存**，回测仍可进行；否则会提示连接失败。删除 `.cache` 可强制重新拉取数据。

## 常见问题

### 提示「缺少 AKShare」

在项目根目录执行：`python -m pip install -r requirements.txt`

### 提示「行情数据暂时连接失败，且没有可用缓存」

- 检查本机网络与 AKShare 数据源是否可访问。
- 缩小年份区间或换一只股票试一次，以生成缓存。
- 东方财富接口失败时，程序会自动尝试新浪等备用接口。

### 找不到股票

- 使用 6 位代码，或输入完整/能唯一匹配的中文名称。
- 首次解析名称列表需联网下载 `stock-names.json`。

### 端口 8000 被占用

修改 `backend/server.py` 中 `main()` 里的 `("127.0.0.1", 8000)` 为其他端口，并用对应地址访问。

### 回测结果与券商 App 不一致

本工具使用**后复权收盘价**和「每年首个交易日买入」等简化规则，与实盘手续费、分红税、涨跌停无法成交等情况不同，结果仅供**策略对比与教学**，不构成投资建议。

## EdgeOne Pages 部署

项目已配置 EdgeOne 全栈部署结构，本地开发与线上共用同一套 API 逻辑（`backend/api_handlers.py`）。

### 部署相关文件

| 文件 | 作用 |
|------|------|
| `edgeone.json` | 输出目录、构建命令、云函数超时与区域 |
| `cloud-functions/api/index.py` | Flask API（`/api/*`） |
| `cloud-functions/requirements.txt` | 云函数依赖（akshare、flask） |
| `scripts/prepare_edgeone.py` | 构建时将 `backend/` 同步到 `cloud-functions/backend/` |

### 本地 EdgeOne 联调（可选）

```powershell
pip install -r cloud-functions/requirements.txt
python scripts/prepare_edgeone.py
npm install -g edgeone
edgeone login
edgeone pages link
edgeone pages dev
```

浏览器访问 **http://localhost:8088**（前端 + `/api` 云函数同端口）。

### 构建配置（`edgeone.json`，推 Git 后自动生效）

| 字段 | 值 |
|------|-----|
| Build output directory | `frontend` |
| Install command | `python3 -m ensurepip --upgrade && python3 -m pip install -r requirements.txt && python3 -m pip install -r cloud-functions/requirements.txt` |
| Build command | `python3 scripts/prepare_edgeone.py` |

注意：不要使用 `pip install --upgrade pip`，EdgeOne 构建机会因 `pyenv-rehash` 权限失败（exit 126）。

### 上线

**Git 部署**：推送后 EdgeOne 读取 `edgeone.json` 自动构建；控制台 **Build & Deploy** → **Deploy** 重新部署。

**CLI 部署**：

```powershell
py scripts/prepare_edgeone.py
edgeone pages deploy
```
